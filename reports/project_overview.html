2. Hệ thống xử lý ticket như thế nào?

Mỗi ticket đi qua một graph gồm nhiều node. Node đầu tiên làm sạch input, node tiếp theo dùng LLM để phân loại, sau đó hệ thống tự chọn nhánh xử lý.

1. Intake Nhận và chuẩn hóa câu hỏi của người dùng.
2. Classify LLM phân loại: simple, tool, missing_info, risky, error.
3. Route Chọn nhánh phù hợp dựa trên route vừa phân loại.
4. Execute Trả lời trực tiếp, gọi tool, hỏi thêm thông tin hoặc xin approval.
5. Evaluate Kiểm tra kết quả tool, retry nếu gặp lỗi tạm thời.
6. Finalize Ghi audit event cuối cùng và kết thúc workflow.

3. Các loại ticket hệ thống xử lý

Route Ý nghĩa Cách xử lý Ví dụ test
simple Câu hỏi thông thường. LLM tạo câu trả lời trực tiếp. How do I reset my password?
tool Cần tra cứu dữ liệu. Gọi mock tool, đánh giá kết quả, rồi trả lời. Please lookup order status for order 12345
missing_info Câu hỏi quá mơ hồ. Không đoán mò, hỏi lại thông tin cần bổ sung. Can you fix it?
risky Hành động có rủi ro hoặc side effect. Chuẩn bị action, yêu cầu approval, rồi mới xử lý. Refund customer, delete account, send email
error Lỗi hệ thống hoặc timeout. Retry có giới hạn, nếu hết lượt thì dead letter. Timeout failure while processing request

5. Những phần đã làm trong project

✓Hoàn thiện state schema: route, risk, retry, evaluation, approval, logs.
✓Hoàn thiện toàn bộ node xử lý: classify, tool, evaluate, answer, clarify, approval, retry, dead letter, finalize.
✓LLM integration: phân loại bằng structured output và sinh câu trả lời bằng LLM.
✓Graph wiring: mọi route đều đi tới finalize và kết thúc đúng.
✓Retry loop có giới hạn, tránh vòng lặp vô hạn.
✓HITL approval path cho các hành động risky.
✓Metrics JSON, report markdown và HTML summary cho người đọc non-tech.
✓Local real data backend: orders, customers, incidents, policies and approvals.
✓Bonus: SQLite checkpoint, Mermaid graph export, Streamlit UI.

7. Báo cáo, log và file evidence nằm ở đâu?

data/support_records.json Dataset local thật: orders, customers, incidents, policies dùng bởi tool.
src/langgraph_agent_lab/tools.py Tool adapter đọc dataset local và trả kết quả lookup/action/retry thật.
outputs/metrics.json Dữ liệu thống kê test: success rate, retries, approvals, route từng scenario.
reports/lab_report.md Báo cáo markdown theo rubric: kiến trúc, schema, metrics, failure analysis, bonus.
reports/project_overview.html File HTML này, dành cho người không chuyên đọc nhanh toàn bộ project.
outputs/graph.mmd Mermaid diagram thể hiện toàn bộ graph và các route.
app.py Streamlit UI để nhập ticket, approve/reject risky action và xem trace.

Log runtime được lưu trong state qua các field append-only như events, messages, tool_results và errors. UI hiển thị các log này trong tab Trace, Tool results và Raw state.

8. Cách chạy demo

Chạy test và sinh metrics

python -m pytest
python -m langgraph_agent_lab.cli run-scenarios --config configs/lab.yaml --output outputs/metrics.json
python -m langgraph_agent_lab.cli validate-metrics --metrics outputs/metrics.json

Chạy UI

streamlit run app.py --server.address 127.0.0.1 --server.port 8501

Sau đó mở http://127.0.0.1:8501 trong trình duyệt.

Trong UI, mở tab All test scenarios để chạy đủ bộ tình huống.
